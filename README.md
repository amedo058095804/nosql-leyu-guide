# nosql-leyu-guide

一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态 HTML 文件的集中存放与版本管理使用。

## 项目简介

本仓库旨在收集、整理并维护一系列独立的 HTML 页面。这些页面可能涵盖技术文档、学习笔记、工具页面或其他类型的静态内容。通过 Git 进行版本控制，便于回溯历史版本、协作修改以及通过 GitHub Pages 等方式直接部署。

## 目录说明

```
nosql-leyu-guide/
├── index.html          # 仓库主页（可选）
├── pages/              # 存放所有独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：主要归档目录，每个 HTML 文件为一个独立页面。
- `assets/`：存放页面共享的样式、脚本及图片资源。
- 根目录下的 `index.html` 可作为仓库的导航页或入口文件。

## 页面归档说明

- 每个 HTML 文件应尽量自包含，必要时可引用 `assets/` 中的公共资源。
- 文件命名建议使用小写字母、数字及连字符，避免空格和特殊字符。
- 如需添加新页面，请直接放入 `pages/` 目录，并在必要时更新导航或索引。

## 维护说明

- 欢迎提交 Pull Request 添加或修改页面内容。
- 提交前请确保 HTML 语法基本正确，并检查资源引用路径。
- 不建议在页面中嵌入外部跟踪脚本或营销内容。
- 如需删除或移动页面，请同时更新相关引用。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 中添加或编辑 HTML 文件。
3. 本地预览后提交变更。
4. 若启用 GitHub Pages，可直接将 `pages/` 或根目录设置为发布源。
